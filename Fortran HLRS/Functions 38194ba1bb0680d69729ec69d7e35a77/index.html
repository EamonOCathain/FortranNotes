    Functions¶

    Functions¶

    A function is a procedure that takes one or more inputs, performs a calculation, and returns a single value.

    Functions are useful for representing mathematical relationships and calculations that can be reused throughout a program.

    Unlike subroutines, functions are called as part of an expression and return a value directly.

    Basic Structure¶

    General form:

    function function_name(argument_list) result(result_name)

    implicit none

    ! declarations

    ! calculations

    result_name = ...

end function function_name

    A function is used like a variable:

    y = function_name(x)

    Example¶

    function square(x) result(y)

    implicit none

    real, intent(in) :: x
    real :: y

    y = x**2

end function square

    Called as:

    real :: a

a = square(4.0)

print *, a

    Output:

    16.0

    Arguments¶

    Functions receive information through an argument list.

    function area(length, width) result(A)

    The variables inside the function are called dummy arguments.

    real, intent(in) :: length
real, intent(in) :: width

    The variables used when calling the function are called actual arguments.

    A = area(x, y)

    The names do not need to match.

    length ↔ x
width  ↔ y

    Arguments are matched by position.

    Return Values¶

    A function returns a single value.

    The modern approach uses the result() keyword.

    function square(x) result(y)

    real, intent(in) :: x
    real :: y

    y = x**2

end function square

    The returned value is stored in the result variable.

    Alternative Syntax¶

    Functions can also return a value using the function name itself.

    real function square(x)

    real, intent(in) :: x

    square = x**2

end function square

    This older style is still valid but is less commonly used in modern Fortran.

    The result() syntax is generally preferred because it is clearer.

    Intent Attributes¶

    Function arguments should normally be declared with:

    intent(in)

    because functions typically use inputs without modifying them.

    Example:

    function cube(x) result(y)

    real, intent(in) :: x
    real :: y

    y = x**3

end function cube

    Attempting to modify an intent(in) argument will generate a compiler error.

    Variable Scope¶

    Variables declared inside a function exist only inside that function.

    function square(x) result(y)

    real :: y
    real :: temp

    temp = x**2
    y = temp

end function square

    The variable temp is not accessible outside the function.

    Likewise, variables from the calling program are not automatically available inside the function unless:

    • they are passed as arguments, or
    • they are imported from a module.

    Internal Functions¶

    Functions can be defined inside a program using contains.

    program example

    implicit none

    real :: x

    x = square(3.0)

    print *, x

contains

    function square(n) result(y)

        real, intent(in) :: n
        real :: y

        y = n**2

    end function square

end program example

    Internal functions can only be used within the program that contains them.

    Module Functions¶

    For larger programs, functions are often placed inside modules.

    module maths

contains

    function square(x) result(y)

        real, intent(in) :: x
        real :: y

        y = x**2

    end function square

end module maths

    Used as:

    use maths

a = square(4.0)

    This allows functions to be reused across multiple programs.

    Functions vs Subroutines¶

    Feature Function Subroutine
    Returns a value Yes No
    Called using y = f(x) call procedure(...)
    Typical use Equations and calculations Tasks and updates
    Number of outputs One Zero, one, or many

    Function:

    y = square(x)

    Subroutine:

    call square(x, y)

    When to Use Functions¶

    Functions are best suited for:

    • mathematical equations
    • unit conversions
    • reusable calculations
    • relationships that produce a single result

    Examples:

    density = mass / volume

area = rectangle_area(length, width)

temperature_c = fahrenheit_to_celsius(temperature_f)

    If a procedure naturally returns one value, it is often a good candidate for a function.

    Complete Example¶

    program demo

    implicit none

    real :: radius
    real :: area

    radius = 2.0

    area = circle_area(radius)

    print *, "Area =", area

contains

    function circle_area(r) result(A)

        real, intent(in) :: r
        real :: A

        A = 3.14159 * r**2

    end function circle_area

end program demo

    Output:

    Area = 12.5664

    How It Works¶

    1. The program assigns a value to radius.
    2. The function circle_area is called.
    3. The function computes the area.
    4. The result is returned through the variable A.
    5. The returned value is assigned to area.
    6. The program prints the result.

    Function Evaluation Order and Side Effects¶

    Functions should behave like mathematical functions: they should compute a result without unexpectedly changing other variables.

    No Guaranteed Evaluation Order¶

    In Fortran, the order in which functions are evaluated within an expression is not defined.

    For example:

    a = fun1(x) * fun2(y)

    The compiler may evaluate either function first.

    Therefore, functions should not modify variables that affect other functions appearing in the same expression.

    Logical Expressions¶

    The same rule applies to logical expressions.

    For example:

    if (associated(ptr) .and. ptr == 1) then
    ...
end if

    The compiler is not required to evaluate associated(ptr) before ptr == 1.

    A safer approach is:

    if (associated(ptr)) then

    if (ptr == 1) then
        ...
    end if

end if

    Avoid Side Effects¶

    A side effect occurs when a function changes something other than its returned value.

    Examples include:

    • modifying global variables
    • modifying module variables
    • modifying its arguments

    Functions with side effects can produce unexpected behaviour when compiler optimizations change evaluation order.

    Best Practice¶

    Functions should:

    • depend only on their input arguments
    • return a single result
    • avoid modifying external variables
    • avoid hidden side effects

    This makes code easier to understand, test, and optimize.

    Summary¶

    Statement Purpose
    function name(...) Begin a function
    result(name) Specify the return variable
    end function name End a function
    intent(in) Declare an input argument
    contains Begin internal procedures
    y = function(...) Call a function

    Key Idea¶

    A function is a reusable block of code that performs a calculation and returns a single value.

    Values are passed into the function through arguments, the calculation is performed, and the result is returned to the calling program.
